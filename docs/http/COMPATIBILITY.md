## 二、 重要发现

### 2.1 outputSchema 问题 ⚠️

**现象**: 服务端声明 `outputSchema` 后，部分工具返回错误 `-32602`

**根因**: 阶跃客户端对 `outputSchema` 严格校验，要求必须返回 `structuredContent`

**解决方案**: **删除所有 `outputSchema` 声明**，只使用 `content` 返回文本

**状态**: ✅ 已修复 (2026-03-10)

### 2.2 客户端超时阈值
- 超时阈值: **55~60 秒**
- 建议: 长操作控制在 55 秒以内
- 或使用 progress notification 保持心跳

### 2.3 流式响应限制
**现象**: SSE 中间事件不传递给 LLM

**影响**:
- 进度通知： 只有最终结果可见
- 长操作: 看不到中间步骤

**建议**: 长任务用轮询模式（start_task → get_status → get_result）

### 2.4 tools/list 缓存
**现象**: 修改服务端后客户端仍使用旧的工具列表

**解决**: 修改工具定义后需要重新连接或刷新客户端

## 五、 给服务端开发者的建议
1. **没必要声明 outputSchema** - 除非确定客户端支持 `structuredContent`
2. **工具执行控制在 55 秒以内** - 超过 60 秒会被超时
3. **长任务用轮询模式** - 流式进度不可见
4. **修改工具后刷新客户端** - 存在缓存
5. **充分利用并发** - 客户端支持并行调用

6. **Unicode 安全** - 中文无需额外处理
