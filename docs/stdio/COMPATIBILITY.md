## 与 HTTP URL 模式的差异

| 特性 | NPX (stdio) | HTTP URL |
|------|-------------|----------|
| 认证 | 不需要 | 需要 API Key |
| 网络 | 不需要 | 需要端口 |
| 启动方式 | 客户端启动 | 手动启动 |
| 进度通知 | 可能更好 | 有限支持 |
| 资源开销 | 较低 | 较高 |

## NPX 模式优势

1. **无需认证**: 本地运行，无需 API Key
2. **零配置网络**: 不占用端口
3. **自动启动**: 客户端自动管理生命周期
4. **更简单**: 适合本地测试
